## Problem-Solving Journey (Firecrawl Integration)

### Timeline of Issues and Solutions

| # | Problem | Attempted Solution | Result | Final Solution |
|---|---------|-------------------|--------|----------------|
| 1 | User said use firecrawl skill, not library | Used `firecrawl scrape -f markdown` | ✗ Only navigation data | Use API extract mode |
| 2 | No product data in output | Tried `--only-main-content` | ✗ Same issue | Schema-based extraction |
| 3 | Still no data | Tried `--country US` | ✗ Same issue | JSON schema with types |
| 4 | CLI needs interactive auth | Tried `-k` flag, env var | ✗ Prompts for login | Use curl directly |
| 5 | API error on first call | Had `cache: false` param | ✗ BAD_REQUEST | Remove invalid param |
| 6 | Final working approach | curl + extract + schema | ✓ SUCCESS | Documented in LESSONS.md |

### Key Learnings

1. **firecrawl CLI** has interactive auth that breaks scripts
2. **Standard scrape** returns navigation only (Amazon anti-bot)
3. **Extract mode** with JSON schema works reliably
4. **curl** is more reliable than CLI for automation
5. **API parameters** must match docs exactly (no `cache: false`)

### Decision Tree for Future Reference

```
Need to scrape Amazon?
│
├─ firecrawl CLI?
│  └─ NO: Interactive auth prompts break scripts
│
├─ firecrawl-py library?
│  └─ NO: Requires pip install, user said no
│
├─ requests + BeautifulSoup?
│  └─ NO: Won't bypass Amazon anti-bot
│
└─ firecrawl API via curl with extract mode?
   └─ YES: This is the working approach!
```
